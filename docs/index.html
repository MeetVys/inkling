A tutorial that is itself the tool

Talk to your docs.
They will build themselves.

inkling is a conversational doc builder living inside Claude Code. You speak normally. The agent designs and edits a real folder of HTML, CSS, JavaScript, and data on your disk. Every doc is interactive, computational, aesthetic, and warm — and grows turn by turn as you think out loud.

Type this in Claude Code
Hey, create an HTML doc that explains the Q1 onboarding revamp. Use a flowchart for the funnel and a chart for the drop-off CSV I'm about to paste.

No slash commands. No markup. Just say what you want — the agent does the rest.

Your turn

Start your first doc in two messages

Open Claude Code in any folder. Paste the first prompt. After it replies, paste the second. That's it.

Message 01

Start the doc

Tell the agent what the doc is for. No slash commands. No template names. Just intent.

Hey, create an HTML doc that captures my notes on [topic]. It should be interactive and feel like a notebook, not a memo.
Message 02

Add something real

Paste data, an image, a section, anything. The agent reads your intent and edits the live folder.

Now add a chart from this data.

step,users
A,100
B,72
C,40
Message 03 (optional)

Iterate naturally

Change anything. Move things around. Drop sections. The agent maintains state across turns — you don't have to re-state which doc you mean.

Make the chart use percentages. Add a footer with my name and today's date.
Message 04 (optional)

See it

Ask to view it. The agent opens the rendered index.html in your browser.

Open the doc. I want to see how it looks.
